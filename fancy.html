Requirements for access

It is already assumed, if you have gained access to cia-files🎮 you satisfy the first two requirements. The last of which is what this guide will teach you to do.

What are QR codes, exactly?

Originally a matrix barcode standard created in 1994 for Japan's automotive industry, it has seen prevalence worldwide due to its capacity to store a lot of information in not a lot fo space. Because of this, and because as images, barcodes are generally considered safe to use in data exchanges (as they are only images after all) it is the standard we use in Dandelion Way as a means of distributing game images packed as CIA files.

Mind, that doesn't mean the QQR coe is for the whole game; These codes are foe file locations where you can receive a title; these codes to not give the full game, only a reference to it. Once received, it can then be installed using FBI in your 3DS, or it can be received directly through FBI per recent updates.

How to use

There are a variety of ways to handle this, so let's go over methods for scanning and receiving one step at a time;

Purposing QR codes

There are several options for doing just that, with the easiest of the methods shown first;

  1. Using your console

    Via FBI, you can select the option Remote Install, then select Scan QR code to acquire a title, but it requires your Internet connection last long enough or possess enough data to handle receiving the data. This is best done with a high-capacity or unlimited plan, or via a connection you know won't be penalized for its use.

  2. Using a web service

    Various web services exist which allow you to right-click (or control-click) a QR code, as they are just pictures to most utilities, though web services will convert the data the image contains into a web address you can then access through your web browser.

  3. Using a mobile handset camera

    The rear-ward facing camera on most mobile handsets ("Smartphones") usually work pretty well for this task, with a multitude of applications designed to scan various codes. However the problem with that is sometimes the camera sensor receives too much light, thus the image ebcomes blown out and unusable. The one primary advantage of this method is it's more socially-acceptable to whip out your phone rather than your 3DS, and some code scanning apps may also have a decode history you can scroll through to download on-the-go or using another Internet connection.

Transferring contents to your console's external storage

If you cannot use your console to scan the code, then you will have to do this. there are a few ways this can be done;

  1. Using a device's internal (micro) SD card interface

    If you use a new 3DS or old 3DS with a microSD card adapter, you can use just about anything with a microSD card interface, however if you are using a full-size SD card in your old 3DS, your options are limited to tablets and desktop PCs with a full-size SD card interface. This is the fastest way to transfer an image, however you also need to be around the device you are using for this to work.

  2. Using an external interface

    If your smartphone lacks an SD card interface but so happens to have USB on-the-go support, you can use USB OTG adapters to transfer data onto a (micro)SD card. Some laptops are also not including SD card interfaces so if your PC was not made during SecureDisk's golden era where it reigned supreme over compact flash and was standard functionality, you will need to use a USB host controller on-board or externally via ExpressCard* to transfer CIA images to your console's SD card.

    Yes, I know ExpressCard isn't even standard anymore, this information was included for completeness, and there may also be SD card adapters for older laptops with the ExpressCard interface, thus making the USB interface not necessary.

  3. Via File Transfer Protocol

    Once modified, a console's external storage can be manipulated via FTP, rather than CIFS which Microsoft depreciated so long ago. If you can use microSD Management on a new 3DS, go for it but if not because your system disallows it, or you don't have a system which even offers that as an option then you will need an FTP client, such as FileZilla (for lack of other cross-system compatible examples) and an FTP server on your 3DS, such as FTPD.

Installing the game

This is fairly simple, and both options can be done via FBI;

  1. Remote install

    If you chose to scan a QR code, you're already set; once received, FBI will install the game via its CIA image for you.

  2. From external storage

    If you had to use a separate device so the CIA file can be put onto the 3DS' external storage, then selecting the SD option will allow yout o install a CIA file. It does not matter where the image is, so long you are able to find it. You can even put a whole bunch of images onto an SD card and use the <Current directory> option to install them all at once; so long a CIA image is in there, options for handling CIA images are presented.

How to generate

Have a game you'd like to share? Want to make it easier for your peers to receive? FOllow these steps first;

  • Find a game you want to share
  • Click through whatever for-profit shorteners are used
  • Right-click on a file's download button and copy its link.

If you cannot share a game directly, feel free to re-upload it! Once you have a URL which is a direct link to the CIA file, you can then codify it using a QR code generator of your choice, however, there are several rules regarding it;

  1. Avoid fancy effects; keep 'em plain
  2. Do not attempt to personalize with your present avatar, 2D game icon or game logo
  3. Do not make them dynamic; there's no need to add another layer of obscurity

Once you have your QR code, you might notice it's a bit large around the edges; this is a "Safe zone" that allows you to put a digital camera's viewfinder in place while taking a picture for decoding, however as some websites had proven previously it doesn't need to be so big, so to shrink it up use a utility like the GNU Image Manipulation Program.

Generation parameters

For legibility, the QR code should have low error tolerance (as these are not being printed) and rendered at a medium size. Any larger and they would take up far too much space, any higher error correction will yield a more dense image; larger in size, but occupying the same space which may make scanning more difficult.

Touching up in GIMP

This is very simple to do for even novice users of the program to figure out on their own, but with GIMP (or GIMP portable) installed you can do the following;

  1. Zoom in and measure one of the blocks with box select
  2. Measure again, this time from the outer edge of the image to the outer edge of the QR data
  3. Subtract from the outer border measure the block measure
  4. Use box select to select all, or invoke select all with pressing A while holding [Ctrl]
  5. From the Select menu, choose to shrink selection
  6. Shrink with the sum received from the subtraction equated (and maybe multiply by two, whatever you like)
  7. Cut the selection, then paste as a new image by pressing V while holding [Shift] and [Ctrl]
  8. Export the image, as a format Discord accepts

Sound complicated? It's not, really; If the gap on all sides is 22px, and the block size is 5px, you can then shrink the selection by 17px and shrink the code by 34px on all sides. This effort not only saves on space, but also saves on disk space when storing them en masse.

Other space-saving measures

If you'd like to store a whole bunch of these, there are other space-saving measures you can take while editing the image which may be worthwhile, potentially shrinking the file size to below 1 megabyte each;

  • Changing the image mode from RGB to indexed, 1-bit
  • Enabling none of the options in PNG format
  • JPEG compression may also help if you crank that up to max
  • (2.10 only) Saving as 8bpc GRAY and not saving extra data with the image

All of this effort means if in the future you want to share any of these images with friends on other platforms, or if you are just hoarding these reference links to other games for yourself you can save a whole bunch of space over time with following the above advice.

Sharing the code(s)

When sharing your QR code in Discord, if you only have one then you can just paste from the clipboard of whatever page or program you copied the image from and append it with the following text in the image sharing dialogue;

**$TITLE ($R)**

If whatever title you are sharing has updates and DLC, it is recommended to share those in Discord for Mobile, as multiple images shared there will show as one big post for an end-user on the desktop version. This is also important for quality-of-life improvements shown in the next section. When sharing through mobile use the following for message input:

**$TITLE ($R; $FILE1, $FILE2, $FILE3)**

Fill in the following information by adjusting the variables as shown;

  • $TITLE: Name of the title you are sharing
  • $R: Title's region (USA, EUR, JPN)
  • $FILE1: The title itself (usually the only image)
  • $FILE2: The title's secondary content (Usually an update)
  • $FILE3: The title's tertiary content (usually DLC)

If you are using Discord for mobile and (transferred your) saved images to your mobile, then using your handset's file manager (if applicable) to rename the images can help influence sharing order. From what has been observed, Discord will always share files with the highest number first in its name, so to brute-force the share order you can rename as shown;

  • titleGame.02.png
  • titleUpd.01.png
  • titleDLC.00.png

Using the above advice, your contribution will not need modification for it to exist comfortably.

Make codes easier to read

Sometimes your display sucks. Sometimes your mobile device's camera is trash. Knowing the console's IP

As follows, perform these steps to enable reception of remote inputs and reveal the console's IP address:

  1. Hold L and Down while pressing Select to cue Luma3DS' Rosalina menu.
    • If you had changed this key combination since, use whatever it was you assigned it to.
  2. Select Miscellaneous options.
  3. Se;ect Start InputRedirection.
  4. Press B to back out until you see the console's IP address.

With the IP address noted, you can begin installing utilities on the PC which can utilize this information to connect with the console.

Acquiring software for remote operation

Both devices needs to be on the same network for any network-based utilities to function.
  1. Download the latest version of Input Redirection Client.
  2. Download the latest version of Snickerstream.
  3. Either version of BootNTR Selector from here.

Input Redirection

No XInput controller? No problem! Expand to see why.

If you lack an XInput-compatible controller such as those for Xbox 360 or Xbox One to use with the redirection client shown above, you're in luck; there exists software which can make any controller compatible.

  1. Download the 32-bit version of DacoTaco's Xbox 360 controller emulator (hereon: x360ce).
  2. Extract, then run the software where it had been extracted.
    • You can also extract x360ce into the directory whose program you want to use it with.
  3. Open x360ce, then permit it to find a functional preset for your controller.
  4. If that doesn't work straight away, end its task, then select Auto in the General tab.
  5. Alternatively, look in the Controller Settings tab to find a functional preset matching your hardware.
  6. If applicable, copy files x360ce.ini and xinput1_3.dll where InputReciredtionClient-Qt.exe is.
  7. Rename xinput1_3.dll to xinput1_4.exe.
    • Alternatively, create a hardlink for xinput1_3.dll named xinput1_4.dll if the software resides on an NTFS partition.
  8. Run InputRedirectionClient-Qt.exe. If successful, you should hear a tone to confirm x360ce is emulating XInput.

Video casting

Some items may simply not work, and launching certain titles while casting may crash the console.
Crashes will not permanently disable the console; simply hold down the power button until the lights go out and power it back on.
  1. On the 3DS, open BootNTR selector.
  2. Go along with default everything, and select version 3.6.
  3. Return to home menu.
  4. Launch Snickerstream.
    • In doing this, a new settings.ini file is made where the executable exists. These settings may be of interest:
      • IpAddr: The default console IP address Snickerstream will look for.
      • Framelimit: Add this in a new line to specify the maximum number of casted frames.

  6. In the Preset section, specify Best framerate to test, initially.
  7. Choose to connect; If it worked, scale video with [Up] and [Down] on the keyboard.
  8. When finished streaming, press [Esc] to exit.
